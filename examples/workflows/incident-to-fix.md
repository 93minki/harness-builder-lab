# Workflow: Incident To Fix

## 목적

모호한 버그 리포트를 재현 가능한 문제 정의로 바꾸고,
위험한 수정을 하기 전에 승인 경계를 분명히 하며,
수정 후 리뷰어에게 필요한 요약까지 준비한다.

## 단계

1. 입력 수집
   - 버그 리포트, Sentry 링크, 최근 PR, 환경 정보를 모은다.
2. 재현 시도
   - 읽기 전용 조사와 로컬 테스트를 통해 증상을 재현한다.
3. 원인 국소화
   - 관련 파일과 의심 로직을 좁힌다.
4. 승인 경계 정의
   - 어떤 수정은 바로 가능하고 어떤 수정은 승인 필요한지 정리한다.
5. 수정 또는 초안 생성
   - 승인 범위 안이면 패치 초안을 만든다.
   - 아니면 수정 계획과 승인 요청 문구를 만든다.
6. 핸드오프 준비
   - 리뷰 요약, 검증 포인트, 후속 작업을 남긴다.

## 성공 조건

- 재현 단계가 문장으로 설명 가능하다
- 의심 파일/모듈이 좁혀졌다
- 승인 필요 작업이 분리되었다
- 리뷰어가 바로 볼 수 있는 요약이 준비되었다
