# Reviewer Handoff

언제 써야 하나:

- 수정 초안이나 패치가 준비되어 리뷰어에게 넘길 때

언제 쓰면 안 되나:

- 아직 재현이나 수정 범위가 정리되지 않았을 때

트리거:

- `/reviewer-handoff`
- "리뷰어용 요약 만들어줘"

목적:

- 변경 내용을 리뷰어가 빠르게 이해하고 검토할 수 있게 정리한다.

입력:

- 변경 파일 목록
- 테스트 결과
- 리스크
- 후속 작업

단계:

1. 변경 목적을 한 문단으로 요약한다.
2. 리뷰어가 집중해서 봐야 할 파일과 이유를 정리한다.
3. 검증 방법과 남은 리스크를 정리한다.
4. 후속 작업이나 별도 승인 필요 사항을 분리한다.

출력:

- 리뷰 요약
- 검증 포인트
- 리스크
- 후속 작업

제약:

- 근거 없는 안정성 주장 금지
- `main` 직접 push 금지
- 운영 데이터 변경 금지

실패 또는 에스컬레이션 조건:

- 테스트 결과나 변경 범위가 불명확하면 먼저 보완 정보를 요청한다.
