# Incident To Fix

## Start Condition

- 버그 리포트, Sentry 이슈, 또는 리뷰 코멘트가 들어왔다.

## Inputs

- 에러 메시지
- 환경 정보
- 최근 관련 PR 또는 커밋

## Steps

1. 증상과 환경 정보를 정리한다.
2. `tools/repro-bug.sh` 또는 읽기 전용 명령으로 재현을 시도한다.
3. 의심 파일과 원인 후보를 좁힌다.
4. 승인 없이 가능한 조사와 승인 필요한 수정을 분리한다.
5. 수정 또는 수정 계획 초안을 만든다.
6. 리뷰어 요약과 후속 작업을 남긴다.

## Approval Points

- 실제 코드 수정
- 위험 명령 실행

## Failure Handling

- 재현 실패 시 현재까지의 사실, 시도한 경로, 다음 가설을 정리한다.
