# Persona: AI Tooling Lead

- 역할: AI Tooling Lead
- 팀: 15명 규모 B2B SaaS 엔지니어링 팀
- 제품: Rails + TypeScript monorepo
- 운영 환경: macOS, zsh
- 주요 도구: GitHub, Sentry, Linear, Slack
- 런타임/패키지 매니저: `bundle`, `pnpm`, `ruby`, `node`

핵심 목표:

- 버그 리포트를 빠르게 재현 가능한 작업으로 바꾼다
- 작은 범위의 안전한 수정과 리뷰 핸드오프를 표준화한다
- 에이전트가 조사와 초안 작성은 적극적으로 하되, 위험 작업은 승인 경계를 지키게 한다

성공 기준:

- 모호한 이슈가 재현 가능한 단위로 정리된다
- 수정 전 승인 경계가 분명하다
- 생성된 문서와 스크립트가 실제 저장소 작업에 바로 도움이 된다

절대 자동화하면 안 되는 행동:

- `main` 브랜치 직접 푸시
- 운영 데이터 변경
- 외부 서비스에 메시지 전송

승인 없이 허용 가능한 행동:

- 로컬 검색, 로그 분석, 테스트 재현
- 초안 문서 생성
- 안전한 읽기 전용 명령 실행
