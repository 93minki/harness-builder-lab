# Examples

이 폴더는 `harness-builder` 스킬로 어떤 결과물을 만들 수 있는지 보여주는 공개 예시 모음이다.
사용자가 스킬을 설치하는 데 이 폴더가 필수는 아니다.
실제 설치에 필요한 파일은 `.claude/skills/harness-builder/SKILL.md` 하나다.

목적:

- 인터뷰 답변이 어떤 식으로 정리될 수 있는지 보여준다
- 플랫폼별 결과물이 어떤 형태로 나올 수 있는지 보여준다
- 사용자가 스킬의 목표 품질을 빠르게 이해하도록 돕는다
- 유지보수 시 예시 기준으로 활용할 수 있다

구성:

- `persona/`: 사용자 역할과 인터뷰 답변
- `workflows/`: 반복 업무 설명
- `expected/`: 플랫폼별 기대 출력 샘플
- `checks/`: 예시를 검토하거나 유지보수할 때 참고할 체크리스트

사용 방법:

1. `persona/interview_answers.md`를 보면 어떤 답변을 주면 되는지 감을 잡을 수 있다.
2. `expected/` 하위 폴더를 보면 플랫폼별 산출물 예시를 볼 수 있다.
3. `checks/` 는 사용자가 설치할 때 필요한 파일이 아니라, 예시를 검토하거나 유지보수할 때 참고하는 문서다.
